Unit I: Java Fundamentals

1. What is the difference between JDK, JRE, and JVM?

JVM (Java Virtual Machine): Abstract machine that executes bytecode. Provides runtime environment.

JRE (Java Runtime Environment): JVM + libraries + other components needed to run Java programs.

JDK (Java Development Kit): JRE + development tools (compiler, debugger) needed to develop Java programs.

Relationship: JDK &superset; JRE &superset; JVM

2. Why is Java called platform-independent?

Java code is compiled to bytecode (.class files), not machine code. This bytecode runs on any platform with a JVM. "Write Once, Run Anywhere" (WORA).

3. Explain the four pillars of OOP.

Encapsulation: Bundling data and methods together, hiding internal state using access modifiers.

Inheritance: Creating new classes from existing ones using extends. Promotes code reuse.

Polymorphism: One interface, multiple implementations. Method overloading (compile-time) and overriding (runtime).

Abstraction: Hiding complex implementation details, showing only essential features via abstract classes/interfaces.

6. What are access modifiers in Java?

private: Accessible only within the same class.

default (no modifier): Accessible within the same package.

protected: Accessible within same package + subclasses in other packages.

public: Accessible from anywhere.

7. Explain the difference between == and equals() in Java.

== operator: Compares references (memory addresses) for objects, values for primitives.

equals() method: Compares content/values. Can be overridden to define equality logic.

Example: Two String objects with same content: == returns false, equals() returns true.

8. What is the String pool?

A special memory area in heap where Java stores String literals. When you create a String literal, JVM checks the pool first. If exists, returns reference to existing String; otherwise creates new one. This optimizes memory by avoiding duplicate String objects.

10. What is a constructor? Types of constructors?

Constructor is a special method called when object is created. Same name as class, no return type.

Default constructor: No parameters. Provided by compiler if none defined.

Parameterized constructor: Takes parameters to initialize object with specific values.

Copy constructor: Creates object by copying another object (not built-in, user-defined).

11. What is the difference between this and super keyword?

this: Refers to current object. Used to access current class members, call current class constructor.

super: Refers to parent class. Used to access parent class members, call parent class constructor.

12. What is static keyword in Java?

Static members belong to class, not instances. Shared across all objects.

Static variable: One copy shared by all instances.

Static method: Can be called without object. Cannot access non-static members directly.

Static block: Executes once when class is loaded.

13. What is final keyword in Java?

final variable: Cannot be changed once initialized (constant).

final method: Cannot be overridden by subclasses.

final class: Cannot be extended (no inheritance).

14. What are wrapper classes?

Classes that wrap primitive types as objects. Required for collections and generics.

int → Integer, char → Character, double → Double, boolean → Boolean, etc.

Autoboxing: Automatic conversion from primitive to wrapper.

Unboxing: Automatic conversion from wrapper to primitive.

15. What is garbage collection?

Automatic memory management. JVM automatically reclaims memory from objects that are no longer referenced. Runs in background, cannot be forced (only requested via System.gc()). Helps prevent memory leaks.

16. What is the difference between String, StringBuilder, and StringBuffer?

String: Immutable. Every modification creates a new object. Thread-safe by immutability.

StringBuilder: Mutable. More efficient for concatenation. Not thread-safe.

StringBuffer: Mutable and synchronized. Thread-safe but slower than StringBuilder.

Use case: String for fixed values, StringBuilder for single-threaded string manipulation, StringBuffer for multi-threaded scenarios.

17. What is method overloading and what are its rules?

Method overloading: Same method name, different parameters in same class. Compile-time polymorphism.

Rules:

  • Must have different parameter list (number or types)
  • Return type can be different but doesn't contribute to overloading
  • Access modifiers can be different
  • Can throw different exceptions
18. What is method overriding and what are its rules?

Method overriding: Subclass provides specific implementation of parent class method. Runtime polymorphism.

Rules:

  • Must have same signature as parent method
  • Return type must be same or covariant
  • Access modifier cannot be more restrictive
  • Cannot override static, final, or private methods
  • @Override annotation recommended
19. What is the difference between == and equals() method?

== operator: Compares references (memory addresses) for objects. For primitives, compares values.

equals() method: Compares content/values of objects. Defined in Object class, can be overridden.

Example: String s1 = "Hello"; String s2 = new String("Hello");
s1 == s2 returns false (different objects)
s1.equals(s2) returns true (same content)

20. What is the purpose of the finalize() method?

Called by garbage collector before destroying object. Used for cleanup activities (close files, release resources).

Note: Not guaranteed to be called. Deprecated in Java 9. Use try-with-resources or cleaner API instead.

22. What is constructor chaining?

Process of calling one constructor from another constructor using this() or super().

this(): Calls another constructor in same class. Must be first statement.

super(): Calls parent class constructor. Must be first statement.

Use case: Avoid code duplication, initialize object in different ways.

23. What is the difference between break and continue?

break: Exits the loop completely. Control moves to statement after loop.

continue: Skips current iteration. Control moves to next iteration.

Both can be used with labels to break/continue outer loops in nested structures.

24. What is instanceof operator?

Tests whether an object is an instance of a specific class or its subclasses.

Syntax: object instanceof Class

Returns: true if object is instance of Class, false otherwise

Use case: Type checking before casting, avoiding ClassCastException

Unit II: Exception Handling, I/O, Multithreading

1. What is the difference between checked and unchecked exceptions?

Checked Exceptions: Must be handled (try-catch) or declared (throws). Checked at compile time. Examples: IOException, SQLException.

Unchecked Exceptions: Not required to handle. Runtime exceptions. Examples: NullPointerException, ArrayIndexOutOfBoundsException.

2. Explain the difference between throw and throws.

throw: Used inside method to explicitly throw an exception. Example: throw new IOException("Error");

throws: Used in method signature to declare exceptions. Example: void read() throws IOException

3. What is the purpose of finally block?

Finally block always executes regardless of whether exception occurs. Used for cleanup operations like closing files/connections. Executes even if try block has return statement. Only skipped if System.exit() is called or JVM crashes.

5. What is the difference between process and thread?

Process: Independent program with its own memory space. Processes don't share memory. Heavy-weight.

Thread: Lightweight subprocess within a process. Threads share memory space. Thread creation is faster.

6. Explain the thread lifecycle.

New: Thread created but not started.

Runnable: Thread ready to run, waiting for CPU.

Running: Thread executing.

Blocked/Waiting: Thread waiting for resource/condition.

Terminated: Thread execution completed.

7. What is synchronization? Why is it needed?

Synchronization controls access to shared resources in multithreaded environment. Prevents race conditions where multiple threads modify shared data simultaneously, leading to data inconsistency. Achieved using synchronized keyword on methods or blocks.

8. What is deadlock?

Deadlock occurs when two or more threads are blocked forever, each waiting for the other's lock. Example: Thread A holds Lock1, waits for Lock2; Thread B holds Lock2, waits for Lock1. Prevented by acquiring locks in same order.

9. How to create a thread in Java?

Method 1: Extend Thread class and override run() method.

Method 2: Implement Runnable interface and pass to Thread constructor.

Method 3: Use ExecutorService (preferred for thread pools).

Implementing Runnable is preferred as it allows extending other classes.

11. What is try-with-resources?

Statement that automatically closes resources after use (Java 7+). Resources must implement AutoCloseable. No need for explicit finally block to close resources. Example: try (FileReader fr = new FileReader("file.txt")) { ... }

12. What is the difference between FileInputStream and FileReader?

FileInputStream: Reads raw bytes. Used for binary files (images, audio).

FileReader: Reads characters using platform default encoding. Used for text files.

13. What is BufferedReader? Why use it?

BufferedReader reads text from character stream with buffering for efficiency. Reduces I/O operations by reading chunks into buffer. Provides readLine() method for line-by-line reading. Faster than reading character by character.

14. What is serialization?

Converting object state to byte stream for storage or transmission. Class must implement Serializable interface. Use ObjectOutputStream to write, ObjectInputStream to read. transient keyword excludes fields from serialization.

15. What is volatile keyword?

Ensures variable is read from main memory, not thread's local cache. Guarantees visibility of changes across threads. Does not provide atomicity - use synchronized or AtomicInteger for atomic operations.

Unit III: Modern Java Features

1. What is a functional interface?

An interface with exactly one abstract method. Can have default/static methods. Marked with @FunctionalInterface annotation (optional but recommended). Used as target for lambda expressions. Examples: Runnable, Comparator, Predicate.

2. What is a lambda expression?

Concise syntax for implementing functional interfaces. Syntax: (parameters) -> expression or (parameters) -> { statements }. Enables functional programming style. Example: (a, b) -> a + b

3. What is a method reference?

Shorthand for lambdas that call existing methods. Uses :: operator. Four types:

Static: Class::staticMethod

Instance of object: object::instanceMethod

Instance of class: Class::instanceMethod

Constructor: Class::new

4. What is the Stream API?

API for processing collections in functional style. Key features:

  • Sequence of elements supporting aggregate operations
  • Intermediate operations (filter, map) return stream
  • Terminal operations (collect, forEach) produce result
  • Lazy evaluation - only computes when needed
  • Supports parallel processing
5. Explain default methods in interfaces.

Methods with implementation in interfaces (Java 8+). Use default keyword. Allow adding new methods to interfaces without breaking existing implementations. Can be overridden by implementing classes.

6. What is the var keyword?

Local variable type inference (Java 10+). Compiler infers type from initialization. Example: var list = new ArrayList<String>();. Only for local variables with initializers. Not a keyword, but reserved type name.

7. What are Records in Java?

Immutable data carriers (Java 16+). Automatically generate constructor, getters, equals(), hashCode(), toString(). Example: record Person(String name, int age) {}. Cannot extend other classes but can implement interfaces.

8. What is try-with-resources?

Automatic resource management (Java 7+). Resources declared in try parentheses are automatically closed. Resource must implement AutoCloseable. Example: try (FileReader fr = new FileReader("file.txt")) { ... }

9. What is Optional class?

Container object that may or may not contain a non-null value (Java 8+). Helps avoid NullPointerException. Methods: of(), ofNullable(), isPresent(), ifPresent(), orElse(), orElseGet(), orElseThrow().

10. What are the built-in functional interfaces in Java?

Predicate<T>: Takes T, returns boolean. test() method.

Function<T,R>: Takes T, returns R. apply() method.

Consumer<T>: Takes T, returns nothing. accept() method.

Supplier<T>: Takes nothing, returns T. get() method.

11. What is the difference between intermediate and terminal operations in Streams?

Intermediate: Return a stream, are lazy (not executed until terminal called). Examples: filter(), map(), sorted(), distinct().

Terminal: Produce result or side-effect, trigger pipeline execution. Examples: collect(), forEach(), count(), reduce().

12. What is switch expression (Java 14+)?

Enhanced switch that can return a value. Uses arrow syntax (->). No fall-through. yield keyword for returning value from block. Example: String result = switch(day) { case 1 -> "Monday"; default -> "Other"; };

13. What are Sealed classes?

Classes that restrict which classes can extend them (Java 17+). Use sealed keyword and permits clause. Subclasses must be final, sealed, or non-sealed. Enables exhaustive pattern matching in switch.

14. What are Text Blocks?

Multi-line string literals (Java 15+). Use triple quotes ("""). Preserve formatting and line breaks. No need to escape quotes. Example: String html = """<html><body></body></html>""";

15. What is pattern matching for instanceof?

Combines instanceof check with variable declaration (Java 16+). Example: if (obj instanceof String s) { s.length(); }. Variable s is automatically cast and scoped.

Unit IV: Collections Framework

1. What is the Collections Framework?

Unified architecture for storing and manipulating groups of objects. Provides interfaces (Collection, List, Set, Map), implementations (ArrayList, HashSet, HashMap), and algorithms (sorting, searching). Located in java.util package.

5. What is the difference between Comparable and Comparator?

Comparable: Natural ordering. Class implements Comparable, defines compareTo(). Single sort sequence. Example: String, Integer.

Comparator: External comparison. Separate class implements Comparator, defines compare(). Multiple sort sequences. More flexible.

6. What is an Iterator?

Object to traverse collections one element at a time. Methods: hasNext(), next(), remove(). Obtained via collection.iterator(). Supports safe element removal during iteration. Fail-fast - throws ConcurrentModificationException if collection modified.

7. What is the difference between fail-fast and fail-safe iterators?

Fail-fast: Throws ConcurrentModificationException if collection modified during iteration. ArrayList, HashMap iterators.

Fail-safe: Works on clone of collection. Doesn't throw exception. ConcurrentHashMap, CopyOnWriteArrayList iterators.

8. What is the internal working of HashMap?

HashMap uses array of buckets. Each bucket is a linked list (Java 8+: converts to tree if > 8 nodes). Process: key.hashCode() -> bucket index -> linear search in bucket using equals(). Load factor (0.75) determines when to resize. Resizing doubles capacity and rehashes all entries.

9. What is the difference between List, Set, and Map?

List: Ordered collection, allows duplicates, index-based access. Examples: ArrayList, LinkedList.

Set: Unordered collection (except TreeSet), no duplicates. Examples: HashSet, TreeSet.

Map: Key-value pairs, unique keys. Not part of Collection interface. Examples: HashMap, TreeMap.

10. What is ConcurrentHashMap?

Thread-safe HashMap alternative. Uses segment-based locking (Java 7) or node-based locking (Java 8+). Better performance than Hashtable. Allows concurrent reads and limited concurrent writes. No null keys or values.

11. What is the difference between Queue and Deque?

Queue: FIFO (First-In-First-Out). Elements added at end, removed from front. Methods: offer(), poll(), peek().

Deque: Double-ended queue. Elements can be added/removed from both ends. Can be used as stack or queue.

12. What is PriorityQueue?

Queue implementation where elements are ordered by priority (natural order or Comparator). Not FIFO. poll() returns highest priority element. Based on heap data structure. Does not allow null.

13. How to make ArrayList thread-safe?

Method 1: Collections.synchronizedList(new ArrayList<>())

Method 2: Use CopyOnWriteArrayList (from java.util.concurrent)

Method 3: Use Vector (legacy, not recommended)

15. What is the hashCode() and equals() contract?

If two objects are equal (equals() returns true), they must have same hashCode.

If two objects have same hashCode, they may or may not be equal.

Must override both together when using objects in hash-based collections.

Unit V: Spring Framework

1. What is Dependency Injection (DI)?

Design pattern where objects receive dependencies from external source rather than creating them. Benefits: loose coupling, easier testing, better maintainability. Types: Constructor injection (preferred), Setter injection, Field injection.

2. What is Inversion of Control (IoC)?

Principle where control of object creation/lifecycle is transferred from program to container/framework. Spring IoC container manages object creation, wiring, and lifecycle. Implements DI to achieve IoC.

3. What are the types of Dependency Injection?

Constructor Injection: Dependencies provided via constructor. Preferred - ensures object fully initialized.

Setter Injection: Dependencies set via setter methods. For optional dependencies.

Field Injection: Dependencies injected directly into fields using @Autowired. Not recommended.

4. What is AOP (Aspect-Oriented Programming)?

Programming paradigm for separating cross-cutting concerns (logging, security, transactions) from business logic. Key concepts: Aspect (module), Join Point (method execution), Pointcut (expression matching join points), Advice (action taken).

5. What are the types of Advice in AOP?

Before: Runs before method execution.

After: Runs after method (regardless of outcome).

After Returning: Runs after successful completion.

After Throwing: Runs if method throws exception.

Around: Wraps method; controls if/when method executes.

6. What are Spring Bean scopes?

Singleton: Single instance per container (default).

Prototype: New instance for each request.

Request: New instance per HTTP request (web).

Session: New instance per HTTP session (web).

Application: Single instance per ServletContext.

7. What is @Autowired annotation?

Spring annotation for automatic dependency injection. Can be used on constructors, setters, or fields. Spring finds matching bean and injects it. Use @Qualifier with @Autowired when multiple beans of same type exist.

8. What is Spring Boot? How is it different from Spring?

Spring Boot is a framework on top of Spring for rapid application development. Differences:

  • Auto-configuration reduces boilerplate
  • Embedded servers (Tomcat) - no external deployment
  • Starter dependencies simplify Maven/Gradle config
  • No XML configuration required
  • Production-ready features (health checks, metrics)
9. What is @Component, @Service, @Repository, @Controller?

@Component: Generic stereotype for Spring-managed component.

@Service: Business logic layer. Semantic clarity.

@Repository: Data access layer. Enables exception translation.

@Controller: Web layer. Handles HTTP requests.

10. What is @RequestMapping?

Maps HTTP requests to handler methods. Attributes: value (URL), method (GET/POST), produces, consumes. Shortcuts: @GetMapping, @PostMapping, @PutMapping, @DeleteMapping.

11. What is the Spring Bean lifecycle?

1. Instantiation: Bean object created.

2. Populate properties: Dependencies injected.

3. BeanNameAware, BeanFactoryAware: Set bean name, factory.

4. Pre-initialization: @PostConstruct or init-method.

5. Ready to use: Bean is fully initialized.

6. Destruction: @PreDestroy or destroy-method on shutdown.

12. What is @Transactional?

Annotation for declarative transaction management. Marks method/class for transaction support. Attributes: propagation, isolation, timeout, readOnly, rollbackFor. Applied to service layer methods.

13. What is ApplicationContext vs BeanFactory?

14. What is @RestController?

Combination of @Controller and @ResponseBody. Every method returns data directly (JSON/XML), not view name. Used for REST APIs. No need for @ResponseBody on each method.

15. What is @PathVariable and @RequestParam?

@PathVariable: Extracts value from URI path. Example: /users/{id} with @PathVariable Long id

@RequestParam: Extracts query parameters. Example: /users?name=John with @RequestParam String name
